# RestoreAssist Project Definition of Done

## Purpose
RestoreAssist is done only as a standalone restoration compliance platform that can be evaluated for business-sale readiness on its own merits. It is not considered done because a single feature, brief, migration lane, video lane, or validation task completed.

## Standalone posture
- RestoreAssist remains standalone for business-sale and operational readiness purposes.
- It is not merged into Unite-Group CRM.
- CRM consolidation is explicitly not the completion model for RestoreAssist.
- Completion claims must cite project-level evidence: data model posture, security/readiness gates, local validation, pilot readiness, business-sale readiness, and Founder / Board acceptance.

## Local evidence closed
This file closes the local evidence gap for standalone RestoreAssist business-sale posture.

## Remaining gates
- Founder / Board human acceptance of sale readiness.
- Any production DB, migration, deployment, secret, OP/1Password, Supabase/psql, Stripe/payments, claims/orders, or public-release action requires a separate named approval.
