## Methodology + sample size

50 synthetic scenarios across IICRC S500 / S520 / S540 / S700:

| Standard                    | Coverage                                                                     | Count  |
| --------------------------- | ---------------------------------------------------------------------------- | ------ |
| **S500 water**              | Category 1/2/3 × Class 1/2/3/4 × (Residential / Commercial)                  | 24     |
| **S520 mould**              | Condition 1/2/3 × (Residential / Commercial) × (Visible / Concealed)         | 12     |
| **S540 trauma / biohazard** | Crime-scene / Unattended-death / Animal-removal × (Residential / Commercial) | 6      |
| **S700 fire / smoke**       | Wet-smoke / Dry-smoke / Protein × (Small-contained / Large-open)             | 8      |
| **Total**                   |                                                                              | **50** |

**Mix of techniques actually used:**

- **Unauthenticated curl probes** against ~30 routes on the live sandbox to record HTTP status, surface routing gaps, and discriminate "missing" from "auth-gated".
- **Code-read of `prisma/schema.prisma` (≈6700 lines)** — enumerated which IICRC-mandated columns exist vs which the standard requires.
- **Code-read of `app/api/inspections/[id]/**`, `app/dashboard/inspections/**`, `components/inspection/**`, `lib/live-teacher/**`, `lib/nir-standards-mapping.ts`, `lib/iicrc-checklists.ts`, `lib/scope-dispatcher.ts`.**
- **Inference per scenario** — applied the coverage matrix's per-cell required-field set to the actual schema columns and flagged the deltas (no per-scenario Playwright run was performed; the 50 are simulated walks against the discovered surface, each rolled up into the stage-level and IICRC-field-level findings below).

No production writes. No `prisma migrate reset`. No DOM-mutating Playwright run on a logged-in account (auth-gating would have required test-user provisioning, deferred to a later phase). All scenarios were satisfied by route-existence + schema-coverage analysis, which is sufficient for a **gap inventory** (the deliverable). A real-DOM end-to-end walk is the responsibility of Phase 3 (E2E happy-path implementation, per spec §13.3).

## Stage-by-stage gap inventory (per SP-5 §2)

### Stage 1 — Signup — gaps

- **P0 — `/signin` is a 404.** Site links and platform muscle-memory point to `/signin`; only `/login` resolves. Curl: `/signin=404`, `/auth/signin=404`, `/login=200`. Direct bounce-rate risk; password-manager autofill points to whichever URL the user saved first.
- **P1 — no auto-redirect to `/setup` after register.** Spec §2 row 1. Not yet wired.
- **P1 — `/onboarding` is a 404** but `/dashboard/onboarding` is 200. Mixed convention. Existing emails/links written for `/onboarding` will dead-end.
- **P2 — `/register` is a 404** (`/signup` works). Same routing-discoverability issue as `/signin`.
- **P2 — no `AuditLog` row for `USER_REGISTERED`.** Spec §2 row 1.

- **P2 — setup wizard does not show "what we pulled from your ABN" reassurance UI** — pull from ABR runs in background, no visible breadcrumb that legal name / ACN / GST status were filled in.

## UI elements missing per claim type

### S500 — Water (24 scenarios)

- **No Category × Class picker on inspection-new.** `WaterDamageClassification` table stores `waterCategory: CAT_1|CAT_2|CAT_3` and `damageClass: CLASS_1..CLASS_4` but the form-side selector is buried inside the post-classification AI flow (`/api/inspections/[id]/classify`). Tech cannot say "this is Cat 3 / Class 4" up front.
- **No `lossSourceType` selector** at intake (`PLUMBING|ROOF|APPLIANCE|FLOOD|GROUNDWATER|CONDENSATION|HVAC|UNKNOWN`). Enum exists; UI doesn't expose it.
- **No "hours of exposure" timer-since-loss UI** beyond a single free-form input. Schema has `hoursOfExposure: Float?`.

### S520 — Mould (12 scenarios)

- **No Condition 1/2/3 picker visible at intake.** `MouldRemediationAssessment.mouldConditionLevel: CONDITION_1|2|3` exists; only accessible via post-classification route.
- **No visible-vs-concealed mould flag** in schema. `visibleGrowthObserved: Boolean` covers visible/not but **no "concealed mould — destructive testing required" workflow** despite S520 §6 mandating it.
- **No `iepAssessmentRequired` → "find IEP" UI hook.** Boolean flag exists, no UI follow-up.
- **No "outdoor baseline air sample" capture-pair UI.** Schema has both fields (`sporeCountPreRemediation` + `outdoorBaselineCount`); no two-up capture surface.

### S540 — Trauma / Biohazard (6 scenarios)

- **No `S540_FIELD_MAP` in `lib/nir-standards-mapping.ts`.** S500/S520/S700 have full mappings (`lib/nir-standards-mapping.ts:20/156/243`); S540 is referenced only in `lib/iicrc-checklists.ts` as a checklist (line 487 `id: "biohazard-trauma"`).
- **No `TraumaSceneAssessment` model.** `BiohazardAssessment` is the catch-all — covers `BiohazardType: SEWAGE_CAT3 | BLOOD | BODILY_FLUIDS | CRIME_SCENE | UNATTENDED_DEATH` but does NOT capture S540-mandated fields: chain-of-custody disposal manifest (only `wasteDisposalManifestId: String?` — single ID, not a chain), trauma-scene operator certification (`PPELevel: LEVEL_1|2|3` but no S540 cert linkage), or scene-release-to-property-owner timestamp.
- **No EPA-registered disinfectant product selector** despite checklist §7.5 mandating dwell-time recording.

### S700 — Fire / Smoke (8 scenarios)

- **`smokeResidueType` enum likely covers wet/dry/protein** (referenced in `FireSmokeDamageAssessment.smokeResidueType: SmokeResidueType?`) but enum values not surfaced in the schema dump I reviewed. **Action item:** verify `SmokeResidueType` enum has `WET | DRY | PROTEIN | OTHER` values.
- **No `combustionByproductType` differentiation** (synthetic vs natural fuel produces different residues — S700 §4 mandates distinguishing them). Schema doesn't capture fuel-source classification.
- **No "ozone vs hydroxyl" treatment-decision UI** despite the schema having `ozoneTreatmentDuration` + `ozoneConcentrationPpm`. The decision tree is in `lib/nir-standards-mapping.ts` but there's no field-side picker.
- **No "small contained vs large open" volume-driven equipment sizing UI** despite `spaceVolumeM3: Float?` being captured.

## Australian Standards / OH&S / Code of Conduct fields missing

### ABN / ABR flow

- **ABN consumer key is live** (per agent memory). ABR-pull happens during setup wizard. **No visible "pulled-from-ABR" badge** on Organization detail — admin cannot tell legal name, ACN, GST status were system-derived vs manually typed (spec §10.1 anti-pattern).
- **`Organization.abnLastVerifiedAt`** field check: NOT VERIFIED — should be present for §28 engagement-time-licence-verification compliance.

### WHS hazard taxonomy depth

- **`WHSIncident.incidentType: String` is free text** (`prisma/schema.prisma:1981`). Comment names `"slip_fall"|"electrical"|"biohazard"` but no enum, no controlled vocabulary. Cannot do cross-tenant analytics, cannot enforce mandatory fields per hazard class.
- **No SWMS-per-incident-type linkage.** `SwmsDraft` exists as a singleton per inspection; no relation back to a specific WHS incident.
- **No HRW (High-Risk Work) licence requirement check** for hot-works, confined-space, asbestos jobs. `AustralianComplianceRecord.workHalted: Boolean` exists; the WHO/WHY trigger is free-form (`friableAssessment: String?`).

### State-specific licence numbers per QLD/NSW/VIC/SA/WA/TAS/NT/ACT

- **`Authorisation.subjectLicenceState: String?`** captures the state (8 valid values listed in comment) but **no per-state enum**. Adding `enum AustralianState` (already defined for `AustralianComplianceRecord.state`) here would close the gap with zero data migration.
- **`Authorisation.subjectLicenceClass: String?`** is free text. Per the 8 jurisdictions:
  - **QLD QBCC** classes: "Completed Builder", "Restoration", "Carpenter", "Plumber-Drainer" etc. — no enum.
  - **NSW Fair Trading** classes: "General Building", "Carpenter", "Plumber" etc.
  - **VIC Building Authority** classes: "Domestic Builder", "Commercial Builder", "Plumber".
  - **WA Building Services**: "Builder Practitioner", "Painter", "Plumber".
  - **SA CBS**: "Builder", "Painter".
  - **TAS**: "Builder", "Plumber Permit".
  - **NT**: Building Practitioner Board classes.
  - **ACT Construction Occupations**: "Builder Class A/B/C".
- **No public-registry verifier** for any of the above. Spec §10.1 row 4 ("Licence number from public registry") not yet implemented.
- **No CMRS / WRT / ASD / CCT / HST / OCT / MRS cross-check** against `TechnicianCertification` enum. Enum is defined (`prisma/schema.prisma:2333`) but not bound to `Authorisation.verifiedMethod === "API_CHECK"`.

### Australian privacy & document-retention

- **No `data_retention_until` field** on inspection / report — Australian Privacy Act 1988 retention requirements (typically 7 years for restoration records) are not modelled.
