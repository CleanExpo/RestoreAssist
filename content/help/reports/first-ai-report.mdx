RestoreAssist drafts compliant S500 / S520 / S540 / S700 reports from the data you've captured on the inspection. The model writes; you review and sign off.

> **A report always starts from an inspection.** You don't type a report from a blank page — you create an [inspection](/dashboard/help/getting-started/first-inspection) first, capture photos + moisture readings + scope items, and the report is generated from that evidence. If you opened `/dashboard/reports/new` and there's nothing to generate from, go capture an inspection first; the report flow will then offer it as the source.

## Before you start

You need:

- An inspection in `IN_PROGRESS` status
- At least 4 photos with chain-of-custody hashes attached
- Moisture readings or affected-area measurements logged
- Scope line items added

Without these inputs the **Generate report** button is disabled — the model needs evidence to cite.

## Generate the draft

Open the inspection's detail page and click **Generate report** in the top-right action bar.

A premium task router (`lib/ai/model-router.ts`) picks the model based on claim complexity. Average turnaround is 10–30 seconds. Bring-your-own-key (BYOK) reduces token cost by 60–80% — set your key in **Settings → AI keys** if you want to use your own.

## What the draft contains

The generated report follows the IICRC structure for your claim type. For water it includes:

1. **Property & loss overview** — address, date of loss, cause
2. **Category & class determination** — per `S500:2021 §10` and `§11`
3. **Affected materials inventory** — with photo references
4. **Moisture mapping** — readings table + interpretation
5. **Scope of work** — line items mapped to drying / extraction / antimicrobial steps
6. **Standards cited** — `S500:2021` sections referenced inline

## Review and edit

The draft is a starting point, not a finished document. Read every section. Common edits:

- Tighten loss-cause wording to match what the homeowner told you
- Re-classify category if the model under- or over-called it
- Add narrative on access constraints or hidden moisture

## Save and sign off

Click **Save draft** to keep iterating, or **Submit for sign-off** when the report is ready. Once signed off, the report is immutable and the inspection moves to `COMPLETED`.
